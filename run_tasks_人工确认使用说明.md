# run_tasks.py 人工确认功能使用说明

## 概述

`run_tasks.py` 现在支持人工确认功能，可以在执行 YAML 任务时等待用户确认后再提交表单。

## 使用方法

### 方法一：交互模式（推荐）

```bash
python run_tasks.py
```

执行流程：

```
1. 选择 "2. 执行单个任务"
2. 选择任务编号（如：task_add.yaml）
3. 选择授权配置
4. 是否启用人工确认？输入 y  ← 关键步骤
5. 是否使用无头模式？输入 n
6. 开始执行
```

### 方法二：命令行模式

```bash
# 不支持命令行参数启用确认功能
# 请使用交互模式
```

## 三个确认工具

### 1. confirm_form_content - 内容确认

**触发时机**：Agent 填写完所有字段后

**功能**：显示已填写的内容，等待用户确认

**用户选项**：
- `y` - 内容正确，继续执行
- `n` - 取消任务，停止执行
- `e` - 需要修改，暂停让用户手动修改

### 2. confirm_final_submit - 最终确认

**触发时机**：内容确认通过后

**功能**：提交前的最后一次确认

**用户选项**：
- `yes` - 确认提交表单
- `no` - 取消提交，停止执行

### 3. pause_for_user - 暂停等待

**触发时机**：需要用户手动操作时

**功能**：暂停任务，显示操作指示，等待用户完成

**用户操作**：完成操作后按 Enter 继续

## YAML 任务文件要求

### 不需要修改 YAML 文件！

原始的 `task_add.yaml` 保持不变：

```yaml
name: teambition任务创建-SP03
task: |
  访问teambition地址（https://www.teambition.com/...）
  点击新建任务
  填入任务标题：SP03-测试任务
  选择项目阶段：PVT
  ...
  禁止点击"完成/完成并创建下一个"按钮，等待用户确认
judge_context:
  - 必须访问teambition地址...
max_steps: 20
```

### 自动增强

当你选择启用人工确认时，`run_tasks.py` 会自动：

1. ✅ 读取原始任务描述
2. ✅ 添加确认指令
3. ✅ 注册确认工具
4. ✅ 控制执行流程

增强后的任务描述会自动包含：

```
原始任务描述...

重要：填写完成后的确认流程
1. 填写完所有字段后，使用 confirm_form_content 工具
2. 传入参数 form_summary，包含所有已填写内容的摘要
3. 等待工具返回结果
4. 如果返回 success=False，立即停止任务
5. 如果返回 success=True，使用 confirm_final_submit 工具
6. 只有在返回 success=True 时才能提交

严格禁止：
- 不要在未调用工具的情况下提交
- 不要在用户取消后继续执行
```

## 使用场景

### ✅ 推荐使用人工确认

- 重要表单提交（任务创建、订单提交等）
- 测试环境中验证填写内容
- 首次运行新任务
- 需要人工审核的流程
- 防止误操作的场景

### ❌ 不推荐使用人工确认

- 完全自动化的批量任务
- 已经充分测试的稳定流程
- 不需要人工干预的场景
- 简单的查询任务

## 常见问题

### Q1: Agent 没有调用确认工具？

**原因**：Agent 可能没有理解确认指令

**解决**：
1. 检查任务描述中是否有"禁止提交"等类似指令
2. 增加 `max_steps` 值，给 Agent 更多步骤
3. 在 YAML 中明确说明：
   ```yaml
   task: |
     ...
     重要：填写完成后必须使用 confirm_form_content 工具
   ```

### Q2: 如何跳过确认直接提交？

**方法**：在选择时输入 `n`（不启用人工确认）

```
是否启用人工确认？(y/n): n
```

### Q3: 可以只在特定任务启用确认吗？

**可以**！每次执行任务时都会询问是否启用确认功能。

### Q4: 确认后 Agent 还是提交了？

**检查**：
1. 确认时是否选择了 `y` 或 `yes`
2. 如果选择了 `n` 或 `no`，Agent 应该停止
3. 如果 Agent 继续执行，可能是任务描述不够明确

### Q5: 可以添加更多确认步骤吗？

**可以**！在 `run_tasks.py` 的 `create_confirmation_tools()` 函数中添加新工具：

```python
@tools.action('确认特定字段')
def confirm_field(field_name: str, field_value: str) -> ActionResult:
    print(f"\n确认 {field_name}: {field_value}")
    choice = input("正确吗？(y/n): ").strip().lower()
    
    if choice == 'y':
        return ActionResult(extracted_content="确认正确", success=True)
    else:
        return ActionResult(error="需要修改", success=False)
```

## 技术细节

### 工具注册

```python
# 创建确认工具
tools = create_confirmation_tools()

# 创建 Agent 时传入
agent = Agent(
    task=enhanced_task,
    llm=ChatBrowserUse(),
    browser=browser,
    tools=tools,  # 注册确认工具
)
```

### 任务增强

```python
if use_confirmation:
    enhanced_task = f'''{original_task}
    
    重要：确认流程
    1. 使用 confirm_form_content 工具
    2. 使用 confirm_final_submit 工具
    ...
    '''
```

### 工具返回值

```python
ActionResult(
    extracted_content="结果描述",
    success=True/False,      # 控制是否继续
    error="错误信息",        # 可选
    is_done=True/False,      # 可选，标记任务完成
)
```
