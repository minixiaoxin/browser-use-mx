## 什么是无头模式？

**无头模式（Headless Mode）** 是指浏览器在后台运行，不显示图形界面窗口。

### 形象比喻

- **有头模式** = 你在电脑前打开浏览器，可以看到窗口和操作
- **无头模式** = 浏览器在后台默默工作，你看不到它，但它在执行任务

---

## 两种模式对比

### 📊 详细对比表

| 特性 | 有头模式<br>(headless=False) | 无头模式<br>(headless=True) |
|------|---------------------------|--------------------------|
| **窗口显示** | ✅ 显示浏览器窗口 | ❌ 不显示窗口 |
| **可视化** | ✅ 可以看到操作过程 | ❌ 看不到过程 |
| **调试方便** | ✅ 容易发现问题 | ⚠️ 需要看日志 |
| **资源占用** | 🟡 高（需要GPU渲染） | 💚 低（无渲染开销） |
| **内存占用** | 🟡 较高 | 💚 较低 |
| **执行速度** | 🐢 稍慢 | ⚡ 更快 |
| **CPU使用** | 🟡 较高 | 💚 较低 |
| **截图功能** | ✅ 支持 | ✅ 支持 |
| **录屏功能** | ✅ 支持 | ✅ 支持 |
| **服务器运行** | ❌ 需要图形环境 | ✅ 可以运行 |
| **并发任务** | ⚠️ 受限于显示器 | ✅ 可以大量并发 |
| **适用场景** | 开发、调试、学习 | 生产、自动化、CI/CD |

## 代码示例

### 示例1：基础用法

#### 有头模式（显示窗口）

```python
from browser_use import Agent, Browser, ChatBrowserUse
from dotenv import load_dotenv
import asyncio

load_dotenv()

async def main():
    # 创建浏览器 - 显示窗口
    browser = Browser(
        headless=False,  # 关键：显示浏览器窗口
        window_size={'width': 1920, 'height': 1080},
    )
    
    agent = Agent(
        task='在百度搜索"Python教程"，返回前3个结果',
        llm=ChatBrowserUse(),
        browser=browser,
    )
    
    history = await agent.run(max_steps=10)
    
    print(f"✅ 任务完成！")
    print(f"结果：{history.final_result()}")

if __name__ == "__main__":
    asyncio.run(main())
```

**运行效果：**
- 🌐 会弹出浏览器窗口
- 👀 可以看到自动操作过程
- 🖱️ 看到鼠标移动和点击
- ⌨️ 看到文字输入

---

#### 无头模式（后台运行）

```python
from browser_use import Agent, Browser, ChatBrowserUse
from dotenv import load_dotenv
import asyncio

load_dotenv()

async def main():
    # 创建浏览器 - 后台运行
    browser = Browser(
        headless=True,  # 关键：隐藏浏览器窗口
    )
    
    agent = Agent(
        task='在百度搜索"Python教程"，返回前3个结果',
        llm=ChatBrowserUse(),

    history = await agent.run(max_steps=10)
    
    print(f"✅ 任务完成！")
    print(f"结果：{history.final_result()}")

if __name__ == "__main__":
    asyncio.run(main())
```

**运行效果：**
- ❌ 不会看到浏览器窗口
- ✅ 任务在后台执行
- ✅ 可以正常获取结果
- ⚡ 执行速度更快

### 示例3：批量任务（无头模式）

```python
from browser_use import Agent, Browser, ChatBrowserUse
import asyncio

async def search_keyword(keyword: str):
    """搜索单个关键词"""
    browser = Browser(
        headless=True,  # 后台运行，支持并发
    )
    
    agent = Agent(
        task=f'在百度搜索"{keyword}"，返回第一个结果',
        llm=ChatBrowserUse(),
        browser=browser,
    )
    
    history = await agent.run(max_steps=8)
    return {
        'keyword': keyword,
        'result': history.final_result()
    }

async def batch_search():
    """批量搜索"""
    keywords = ['Python', 'JavaScript', 'Go', 'Rust', 'Java']
    
    # 并发执行多个任务
    tasks = [search_keyword(kw) for kw in keywords]
    results = await asyncio.gather(*tasks)
    
    for r in results:
        print(f"{r['keyword']}: {r['result']}")

asyncio.run(batch_search())
```

**优势：**
- ✅ 无头模式支持大量并发
- ✅ 不受显示器限制
- ✅ 资源占用更少

```python
# 生产时使用
result = asyncio.run(run_task(debug_mode=False))
```

## 常见问题

### Q1: 无头模式下可以截图吗？

**A:** ✅ 可以！无头模式完全支持截图功能。

```python
browser = Browser(headless=True)

agent = Agent(
    task='访问百度首页并截图',
    llm=ChatBrowserUse(),
    browser=browser,
)

history = await agent.run()

# 获取截图
screenshots = history.screenshots()
for i, screenshot in enumerate(screenshots):
    with open(f'screenshot_{i}.png', 'wb') as f:
        f.write(screenshot)
```

---

### Q2: 无头模式下如何调试？

**A:** 使用日志和截图：

```python
import logging

# 开启详细日志
logging.basicConfig(level=logging.DEBUG)

browser = Browser(
    headless=True,
)

agent = Agent(
    task='你的任务',
    llm=ChatBrowserUse(),
    browser=browser,
    generate_gif=True,  # 生成操作过程的GIF
)

history = await agent.run()

# 查看访问的URL
print(f"访问的URL：{history.urls()}")

# 查看错误
print(f"错误：{history.errors()}")

# 查看截图
screenshots = history.screenshots()
```

---

### Q3: 服务器上必须用无头模式吗？

**A:** ✅ 是的。大多数服务器没有图形界面，必须使用无头模式。

```python
# 服务器环境
browser = Browser(
    headless=True,  # 必须
)
```

如果服务器有图形界面（如VNC），也可以使用有头模式。

---

### Q4: 无头模式会影响功能吗？

**A:** ❌ 不会！功能完全相同：

- ✅ 所有浏览器操作
- ✅ JavaScript 执行
- ✅ 截图和录屏
- ✅ Cookie 和存储
- ✅ 文件下载
- ✅ 表单填写

唯一区别是看不到窗口。

---

### Q5: 如何在 Docker 中使用？

**A:** Docker 环境必须使用无头模式：

```python
browser = Browser(
    headless=True,
    chromium_sandbox=False,  # Docker中禁用沙箱
)
```

---

### Q6: 无头模式更快的原因？

**A:** 主要原因：

1. **无渲染开销** - 不需要绘制界面
2. **无GPU计算** - 不需要图形加速
3. **更少内存** - 不需要缓存显示内容
4. **更少CPU** - 不需要处理UI事件

---

### Q7: 如何临时切换模式？

**A:** 使用环境变量：

```python
import os

# 通过环境变量控制
headless = os.getenv('HEADLESS', 'true').lower() == 'true'

browser = Browser(headless=headless)
```

运行时切换：
```bash
# 有头模式
HEADLESS=false uv run python my_script.py

# 无头模式
HEADLESS=true uv run python my_script.py
```

## 最佳实践

### ✅ 推荐做法

1. **开发时用有头模式**
   ```python
   browser = Browser(headless=False)
   ```

2. **生产时用无头模式**
   ```python
   browser = Browser(headless=True)
   ```

3. **使用环境变量控制**
   ```python
   headless = os.getenv('ENVIRONMENT') == 'production'
   browser = Browser(headless=headless)
   ```

4. **调试时临时切换**
   ```python
   DEBUG = True  # 调试时设为 True
   browser = Browser(headless=not DEBUG)
   ```

---

### ❌ 避免的做法

1. **生产环境用有头模式** - 浪费资源
2. **服务器上强制有头模式** - 会失败
3. **不考虑环境差异** - 难以部署
