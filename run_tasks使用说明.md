# 🚀 run_tasks.py 增强版使用说明

## 📋 新增功能

### ✨ 主要改进

1. **授权配置管理** - 支持多种登录方式
2. **自动检测配置** - 自动发现可用的授权配置
3. **配置管理界面** - 创建、删除、测试授权配置
4. **灵活选择** - 执行任务时可选择使用哪个授权配置

---

## 🎯 支持的授权类型

### 1. 无授权（默认）
- 使用全新浏览器实例
- 无登录状态
- 适合不需要登录的任务

### 2. 保存的Cookie
- 使用之前保存的登录状态
- 存储在 `*_data` 目录
- 适合生产环境

### 3. 真实浏览器配置
- 使用本地 Chrome 浏览器的配置
- 保留所有登录状态
- 适合本地开发

---


## 🔐 授权配置管理

### 进入配置管理

```bash
uv run python run_tasks.py
# 选择 "4. 管理授权配置"
```

### 配置管理选项

#### 1. 查看所有授权配置

显示所有可用的授权配置：
- 无授权（默认）
- 保存的Cookie配置
- 真实浏览器配置

**示例输出：**
```

============================================================

1. 🌐 无授权
   类型：none
   说明：使用全新浏览器实例（无登录状态）

   类型：saved_cookies
   说明：保存的Cookie (taobao_data)
   路径：./taobao_data

3. 🔑 Chrome Default
   类型：real_browser
   说明：真实Chrome浏览器 (Default配置)
   配置：Default
```

---

#### 2. 创建新的Cookie配置

创建一个新的授权配置目录，用于保存登录状态。

**步骤：**
1. 输入配置名称（如：`taobao`, `github`, `jd`）
2. 确认创建
3. 系统会创建 `{name}_data` 目录
4. 首次使用时会打开浏览器让你登录

**示例：**
```bash
请输入配置名称（如：taobao, github）: taobao

💡 将创建配置：taobao
   数据目录：./taobao_data

📝 接下来会打开浏览器，请手动登录需要的网站

确认创建？(y/n): y

✅ 配置目录已创建：./taobao_data
💡 现在可以使用此配置执行任务
   首次使用时会打开浏览器让你登录
```

---

#### 3. 删除Cookie配置

删除不再需要的Cookie配置。

**步骤：**
1. 显示所有可删除的配置
2. 选择要删除的配置
3. 输入 `yes` 确认删除

**注意：** 删除后无法恢复！

---

#### 4. 测试授权配置

测试授权配置是否正常工作。

**功能：**
- 选择一个授权配置
- 打开浏览器访问百度首页
- 验证配置是否有效

**用途：**
- 检查Cookie是否过期
- 验证真实浏览器配置是否正确
- 确认登录状态

## 💡 使用场景示例

### 场景1：执行需要登录的任务（淘宝）

**步骤：**

1. **首次使用 - 创建配置**
   ```bash
   uv run python run_tasks.py
   # 选择 "4. 管理授权配置"
   # 选择 "2. 创建新的Cookie配置"
   # 输入名称：taobao
   ```

2. **创建任务文件**
   ```yaml
   # tests/agent_tasks/taobao_search.yaml
   name: 淘宝商品搜索
   task: |
     1. 访问淘宝
     2. 搜索"机械键盘"
     3. 提取前5个商品的名称和价格
   judge_context:
     - 必须访问淘宝
     - 必须搜索"机械键盘"
     - 必须返回5个商品
   max_steps: 15
   ```

3. **执行任务**
   ```bash
   uv run python run_tasks.py
   # 选择 "2. 执行单个任务"
   # 选择任务：taobao_search.yaml
   # 选择授权：Taobao
   # 首次会打开浏览器让你登录淘宝
   # 登录后任务自动执行
   ```

4. **后续使用**
   - 直接选择 Taobao 配置
   - 自动使用保存的登录状态
   - 无需重复登录

---

### 场景2：使用真实浏览器（本地开发）

**适用于：** 你已经在 Chrome 中登录了需要的网站

**步骤：**

1. **在 Chrome 中登录网站**
   - 打开 Chrome
   - 登录淘宝、GitHub 等网站
   - 完全关闭 Chrome

2. **执行任务**
   ```bash
   uv run python run_tasks.py
   # 选择 "2. 执行单个任务"
   # 选择任务
   # 选择授权：Chrome Default
   # 系统会提示关闭 Chrome
   # 确认后执行任务
   ```

3. **优点**
   - 使用真实浏览器配置
   - 保留所有登录状态
   - 保留浏览器扩展

---

### 场景3：批量执行任务

**步骤：**

## 💡 最佳实践

### 1. 配置命名规范

```bash
# 推荐命名
taobao_data      # 淘宝
github_data      # GitHub
jd_data          # 京东
weibo_data       # 微博

# 避免使用

my_config        # 不明确
test             # 容易混淆
```

---

### 2. 定期更新Cookie

Cookie 会过期，建议：
- 每周检查一次登录状态
- 使用 "测试授权配置" 功能验证
- 过期后重新登录

---

### 3. 选择合适的授权方式

| 场景 | 推荐方式 | 原因 |
|------|---------|------|
| 本地开发 | 真实浏览器 | 最方便 |
| 生产环境 | 保存Cookie | 稳定可靠 |
| 临时测试 | 无授权 | 快速简单 |
| 多账号 | 多个Cookie配置 | 灵活切换 |

---

### 4. 安全建议

- ⚠️ 不要提交 `*_data` 目录到 Git
- ⚠️ 添加到 `.gitignore`：
  ```
  *_data/
  ```
- ⚠️ 定期清理不用的配置
- ⚠️ 使用环境变量存储敏感信息

## 🐛 常见问题

### Q1: 为什么找不到授权配置？

**A:** 检查：
1. 是否创建了配置目录
2. 目录名是否以 `_data` 结尾
3. 是否在项目根目录运行

---

### Q2: 真实浏览器配置无法使用？

**A:** 确保：
1. 完全关闭 Chrome 浏览器
2. 检查任务管理器，确保没有 chrome.exe
3. Chrome 路径是否正确

---

### Q3: Cookie 配置失效？

**A:** 可能原因：
1. Cookie 已过期
2. 网站更新了登录机制
3. 需要重新登录

**解决方案：**
1. 删除旧配置
2. 创建新配置
3. 重新登录

---

### Q4: 如何切换不同账号？

**A:** 创建多个配置：
```bash
taobao_account1_data
taobao_account2_data
github_personal_data
github_work_data
```

执行时选择对应的配置即可。
