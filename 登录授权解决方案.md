## 问题说明

### 常见场景
- 🔒 需要登录的网站（淘宝、京东、GitHub等）
- 🎫 需要验证码的网站
- 🔑 需要二次验证（2FA）的网站
- 📱 需要扫码登录的网站

### 核心挑战
每次运行 Browser-Use 都会创建一个**全新的浏览器实例**，没有任何登录状态和Cookie。

## 方案1：使用真实浏览器配置

### 💡 原理
使用你本地已经登录的 Chrome 浏览器配置文件，这样所有的登录状态、Cookie、扩展都会被保留。

### ✅ 优点
- 最简单直接
- 保留所有登录状态
- 保留浏览器扩展
- 保留所有设置

### ⚠️ 注意事项
- 需要完全关闭 Chrome 浏览器
- 只能在本地使用
- 不适合服务器环境

---

### 📝 实现步骤

#### 步骤1：找到你的 Chrome 配置文件路径

**Windows:**
```
C:\Users\你的用户名\AppData\Local\Google\Chrome\User Data
```

**macOS:**
```
~/Library/Application Support/Google/Chrome
```

**Linux:**
```
~/.config/google-chrome
```

#### 步骤2：找到 Chrome 可执行文件路径

**Windows:**
```
C:\Program Files\Google\Chrome\Application\chrome.exe
```

**macOS:**
```
/Applications/Google Chrome.app/Contents/MacOS/Google Chrome
```

**Linux:**
```
/usr/bin/google-chrome
```

#### 步骤3：使用配置文件

```python
from browser_use import Agent, Browser, ChatBrowserUse
from dotenv import load_dotenv
import asyncio

load_dotenv()

async def main():
    # 使用真实浏览器配置
    browser = Browser(
        # Chrome 可执行文件路径
        executable_path=r'C:\Program Files\Google\Chrome\Application\chrome.exe',
        
        # Chrome 用户数据目录
        user_data_dir=r'C:\Users\你的用户名\AppData\Local\Google\Chrome\User Data',
        
        # 使用的配置文件（默认是 Default）
        profile_directory='Default',  # 或 'Profile 1', 'Profile 2' 等
        
        # 显示窗口
        headless=False,
    )
    
    agent = Agent(
        task='访问需要登录的网站，执行操作',
        llm=ChatBrowserUse(),
        browser=browser,
    )
    
    history = await agent.run()
    print(f"结果：{history.final_result()}")

if __name__ == "__main__":
    asyncio.run(main())
```

#### 步骤4：完整示例（Windows）

```python
from browser_use import Agent, Browser, ChatBrowserUse
from dotenv import load_dotenv
import asyncio
import os

load_dotenv()

async def use_real_browser():
    """使用真实浏览器配置"""
    
    # Windows 路径
    chrome_path = r'C:\Program Files\Google\Chrome\Application\chrome.exe'
    user_data_dir = os.path.expanduser(r'~\AppData\Local\Google\Chrome\User Data')
    
    # macOS 路径（如果是 Mac，取消注释）
    # chrome_path = '/Applications/Google Chrome.app/Contents/MacOS/Google Chrome'
    # user_data_dir = os.path.expanduser('~/Library/Application Support/Google/Chrome')
    
    # Linux 路径（如果是 Linux，取消注释）
    # chrome_path = '/usr/bin/google-chrome'
    # user_data_dir = os.path.expanduser('~/.config/google-chrome')
    
    print("⚠️ 重要：请先完全关闭 Chrome 浏览器！")
    input("关闭后按 Enter 继续...")
    
    browser = Browser(
        executable_path=chrome_path,
        user_data_dir=user_data_dir,
        profile_directory='Default',
        headless=False,
    )
    
    agent = Agent(
        task='''
        1. 访问 GitHub
        2. 检查是否已登录
        3. 如果已登录，访问你的个人主页
        4. 返回用户名
        ''',
        llm=ChatBrowserUse(),
        browser=browser,
    )
    
    history = await agent.run(max_steps=10)
    print(f"\n✅ 结果：{history.final_result()}")

if __name__ == "__main__":
    asyncio.run(use_real_browser())
```

## 方案2：保存和加载Cookie

### 💡 原理
第一次手动登录后，保存 Cookie 和存储状态，之后自动加载这些状态。

### ✅ 优点
- 适合生产环境
- 可以在服务器上使用
- 不需要真实浏览器配置

### 📝 实现步骤

#### 步骤1：首次登录并保存状态

```python
from browser_use import Agent, Browser, ChatBrowserUse
from dotenv import load_dotenv
import asyncio

load_dotenv()

async def save_login_state():
    """首次登录并保存状态"""
    
    browser = Browser(
        headless=False,  # 显示窗口，方便手动登录
        user_data_dir='./browser_data',  # 保存到本地目录
    )
    
    agent = Agent(
        task='''
        1. 访问需要登录的网站
        2. 等待用户手动登录
        3. 登录成功后，访问个人主页确认
        ''',
        llm=ChatBrowserUse(),
        browser=browser,
    )
    
    print("🔐 请在浏览器中手动登录...")
    print("⏳ Agent 会等待你完成登录...")
    
    history = await agent.run(max_steps=15)
    
    print("\n✅ 登录状态已保存到 ./browser_data 目录")
    print("💡 下次运行时会自动使用这个登录状态")

if __name__ == "__main__":
    asyncio.run(save_login_state())
```

#### 步骤3：使用 storage_state（更精确）

```python
from browser_use import Agent, Browser, ChatBrowserUse
from browser_use.browser.session import BrowserSession
from dotenv import load_dotenv
import asyncio
import json

load_dotenv()

async def save_storage_state():
    """保存存储状态（Cookie、localStorage等）"""
    
    browser = Browser(
        headless=False,
    )
    
    # 创建会话
    session = BrowserSession(browser_profile=browser)
    await session.start()
    
    print("🔐 请在浏览器中手动登录...")
    input("登录完成后按 Enter 继续...")
    
    # 保存存储状态
    storage_state = await session.get_storage_state()
    
    with open('storage_state.json', 'w', encoding='utf-8') as f:
        json.dump(storage_state, f, indent=2)
    
    print("✅ 存储状态已保存到 storage_state.json")
    
    await session.kill()

async def load_storage_state():
    """加载存储状态"""
    
    # 读取存储状态
    with open('storage_state.json', 'r', encoding='utf-8') as f:
        storage_state = json.load(f)
    
    browser = Browser(
        headless=True,
        storage_state=storage_state,  # 加载状态
    )
    
    agent = Agent(
        task='访问需要登录的网站，执行操作',
        llm=ChatBrowserUse(),
        browser=browser,
    )
    
    history = await agent.run(max_steps=10)
    print(f"✅ 结果：{history.final_result()}")

# 首次运行：保存状态
# asyncio.run(save_storage_state())

# 后续运行：加载状态
asyncio.run(load_storage_state())
```

## 方案3：使用云端浏览器配置

### 💡 原理
使用 Browser-Use Cloud 的配置文件功能，在云端保存登录状态。

### ✅ 优点
- 最简单的生产环境方案
- 自动同步登录状态
- 支持多设备共享
- 内置反爬虫保护

### 📝 实现步骤

#### 步骤1：创建云端配置文件

```bash
# 设置 API Key
export BROWSER_USE_API_KEY=your_key

# 运行配置脚本（会打开浏览器让你登录）
curl -fsSL https://browser-use.com/profile.sh | sh
```

这会：
1. 打开一个浏览器
2. 让你登录需要的网站
3. 保存登录状态到云端
4. 返回一个 `profile_id`

#### 步骤2：使用云端配置

```python
from browser_use import Agent, Browser, ChatBrowserUse
from dotenv import load_dotenv
import asyncio

load_dotenv()

async def use_cloud_profile():
    """使用云端浏览器配置"""
    
    browser = Browser(
        use_cloud=True,  # 使用云端浏览器
        cloud_profile_id='your-profile-id',  # 你的配置ID
        cloud_proxy_country_code='us',  # 可选：使用代理
    )
    
    agent = Agent(
        task='访问需要登录的网站，执行操作',
        llm=ChatBrowserUse(),
        browser=browser,
    )
    
    history = await agent.run(max_steps=10)
    print(f"✅ 结果：{history.final_result()}")

if __name__ == "__main__":
    asyncio.run(use_cloud_profile())
```

## 方案4：人工介入模式

### 💡 原理
在需要登录时暂停，让用户手动登录，然后继续执行。

### 📝 实现代码

```python
from browser_use import Agent, Browser, Tools, ChatBrowserUse, ActionResult
from dotenv import load_dotenv
import asyncio

load_dotenv()

# 创建自定义工具
tools = Tools()

@tools.action('等待用户手动操作')
def wait_for_user(instruction: str) -> ActionResult:
    """
    暂停执行，等待用户手动操作
    
    Args:
        instruction: 给用户的指示
    """
    print(f"\n⏸️ 任务暂停")
    print(f"📝 请执行以下操作：{instruction}")
    input("\n✅ 完成后按 Enter 继续...")
    
    return ActionResult(
        extracted_content="用户已完成手动操作",
        success=True
    )

async def main():
    browser = Browser(
        headless=False,  # 必须显示窗口
    )
    
    agent = Agent(
        task='''
        1. 访问需要登录的网站
        2. 如果需要登录，使用 wait_for_user 工具让用户手动登录
        3. 登录后继续执行任务
        ''',
        llm=ChatBrowserUse(),
        browser=browser,
        tools=tools,
    )
    
    history = await agent.run(max_steps=15)
    print(f"\n✅ 结果：{history.final_result()}")

if __name__ == "__main__":
    asyncio.run(main())
```

## 方案5：自定义登录工具

### 💡 原理
为特定网站编写自动登录逻辑。

### 📝 实现代码

```python
from browser_use import Agent, Browser, Tools, ChatBrowserUse, ActionResult
from dotenv import load_dotenv
import asyncio

load_dotenv()

tools = Tools()

@tools.action('使用保存的账号密码登录')
def auto_login(website: str) -> ActionResult:
    """
    自动登录指定网站
    
    Args:
        website: 网站名称（如 'github', 'taobao'）
    """
    # 从环境变量或配置文件读取账号密码
    import os
    
    credentials = {
        'github': {
            'username': os.getenv('GITHUB_USERNAME'),
            'password': os.getenv('GITHUB_PASSWORD'),
        },
        'taobao': {
            'username': os.getenv('TAOBAO_USERNAME'),
            'password': os.getenv('TAOBAO_PASSWORD'),
        },
    }
    
    if website not in credentials:
        return ActionResult(
            extracted_content=f"未配置 {website} 的登录信息",
            error=f"No credentials for {website}",
            success=False
        )
    
    creds = credentials[website]
    
    return ActionResult(
        extracted_content=f"使用账号 {creds['username']} 登录 {website}",
        success=True
    )

@tools.action('获取2FA验证码')
def get_2fa_code() -> str:
    """获取二次验证码"""
    code = input("\n🔐 请输入2FA验证码: ")
    return code

async def main():
    browser = Browser(
        headless=False,
    )
    
    agent = Agent(
        task='''
        1. 访问 GitHub
        2. 如果需要登录，使用 auto_login 工具
        3. 如果需要2FA，使用 get_2fa_code 工具
        4. 登录后访问个人主页
        ''',
        llm=ChatBrowserUse(),
        browser=browser,
        tools=tools,
    )
    
    history = await agent.run(max_steps=15)
    print(f"\n✅ 结果：{history.final_result()}")

if __name__ == "__main__":
    asyncio.run(main())
```

```python
   browser = Browser(

   ```

## 常见问题

### Q1: 为什么使用真实浏览器时提示"Chrome is already running"？

**A:** 必须完全关闭 Chrome 浏览器才能使用其配置文件。

**解决方案：**
1. 关闭所有 Chrome 窗口
2. 检查任务管理器，确保没有 chrome.exe 进程
3. 再运行脚本

---

### Q2: Cookie 多久会过期？

**A:** 取决于网站设置，通常：
- 短期：几小时到几天
- 长期：几周到几个月
- 永久：需要定期刷新

**解决方案：** 定期重新登录更新状态

---

### Q3: 如何处理验证码？

**A:** 几种方案：
1. 使用云端浏览器（内置反爬虫）
2. 使用人工介入模式
3. 使用验证码识别服务
4. 使用已登录的浏览器配置

---

### Q4: 如何处理扫码登录？

**A:** 使用人工介入模式：
```python
@tools.action('等待扫码登录')
def wait_for_qr_scan() -> str:
    input("请使用手机扫码登录，完成后按 Enter...")
    return "扫码完成"
```
