## 关键特性

### 1. 无需修改 YAML 文件 ✅

`task_add.yaml` 保持原样：

```yaml
name: teambition任务创建-SP03
task: |
  访问teambition地址...
  填入任务标题：SP03-测试任务
  ...
  禁止点击"完成"按钮，等待用户确认
max_steps: 20
```

`run_tasks.py` 会自动添加确认指令！

### 2. 灵活启用/禁用 ✅

每次执行时选择：
- 输入 `y` - 启用人工确认
- 输入 `n` - 跳过确认，直接执行

### 3. 双重确认机制 ✅

- **第一次确认**：检查填写内容是否正确
- **第二次确认**：确认是否提交表单

### 4. 支持修改 ✅

在第一次确认时选择 `e`：
- 暂停任务
- 在浏览器中手动修改
- 修改完成后继续

## 工作原理

### 自动增强任务描述

```python
if use_confirmation:
    enhanced_task = f'''{original_task}

    重要：填写完成后的确认流程
    1. 使用 confirm_form_content 工具
    2. 传入 form_summary 参数
    3. 等待返回结果
    4. 如果 success=False，停止
    5. 使用 confirm_final_submit 工具
    6. 只有 success=True 才能提交
    
    严格禁止：
    - 不要在未确认的情况下提交
    '''
```

### 工具注册

```python
if use_confirmation:
    tools = create_confirmation_tools()
else:
    tools = None

agent = Agent(
    task=task_to_run,
    llm=ChatBrowserUse(),
    browser=browser,
    tools=tools,  # 传入确认工具
)
```

## 使用场景

### ✅ 推荐启用确认

- 重要表单提交（任务创建、订单提交）
- 首次运行新任务
- 测试环境验证
- 需要人工审核的流程
- 防止误操作

### ❌ 不推荐启用确认

- 简单查询任务
- 不需要人工干预

## 常见问题

### Q1: 需要修改 task_add.yaml 吗？

**不需要！** `run_tasks.py` 会自动增强任务描述。

### Q2: 如何启用确认功能？

在执行时选择：
```
是否启用人工确认？(y/n): y
```

### Q3: 可以跳过确认吗？

可以！选择时输入 `n`：
```
是否启用人工确认？(y/n): n
```

### Q4: Agent 没有调用确认工具？

**解决方法**：
1. 增加 `max_steps` 值
2. 在 YAML 中明确说明：
   ```yaml
   task: |
     ...
     重要：填写完成后必须使用 confirm_form_content 工具
   ```

### Q5: 确认后还是提交了？

检查：
- 是否选择了 `yes`（确认提交）
- 如果选择了 `no`，Agent 应该停止
- 如果继续执行，可能是任务描述不够明确

## 技术细节

### 工具返回值

```python
ActionResult(
    extracted_content="结果描述",
    success=True/False,      # 控制是否继续
    error="错误信息",        # 可选
    is_done=True/False,      # 可选
)
```

### 流程控制

- `success=True` → Agent 继续执行
- `success=False` → Agent 停止执行
- `is_done=True` → 标记任务完成
